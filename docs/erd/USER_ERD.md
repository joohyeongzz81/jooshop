# User 도메인 설계

## 개요
회원 정보를 관리하는 엔티티

---

## 테이블: users

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|----------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 회원 ID |
| email | VARCHAR(255) | UNIQUE, NOT NULL | 이메일 (로그인 ID) |
| name | VARCHAR(100) | NOT NULL | 회원 이름 |
| created_at | TIMESTAMP | NOT NULL | 가입일시 |

---

## 비즈니스 규칙

### 1. 이메일 중복 불가
- 동일한 이메일로 중복 가입 불가
- 회원 가입 시 중복 검증 필수

### 2. 필수 입력 항목
- 이메일: 이메일 형식 검증
- 이름: 2-100자 제한

### 3. 생성 시점 자동 기록
- created_at은 회원 생성 시 자동 설정

## 주요 기능

### 1. 회원 가입
- 이메일, 이름 입력
- 이메일 중복 검증
- 회원 정보 저장

### 2. 회원 조회
- ID로 회원 정보 조회
